  • freeTeal pill · price reads Free · RSVP → the venue's event page

Edge cases the component has to survive

Triangle Shows fields are mostly nullable, and an artist can have more than one date. The ticket degrades to whatever's present — it never renders an empty stat or a dangling "with".

Implementation notes

  • State is the API's, verbatim. Map status → pill + CTA with no client heuristics. Unknown or missing status falls back to the on_sale presentation.
  • The CTA links to the venue's event page. Target source_url (the venue's own event page); fall back to ticket_url only when source_url is null. Always render the external-link glyph and name the venue — WXYC hands off to the box office, it doesn't sell tickets.
  • Degrade on nulls. Render a stat only when its field is non-null; the grid auto-fits 1–3 columns and hides when empty (.stats:empty). Drop "with …" when support_artists is null; drop age when age_restriction is null.
  • Price formatting. free → "Free"; min == max → "$22"; else "$22–$25"; both null → omit the stat.
  • Match on a name, guarded. No artist ID upstream — highlight only on a normalized exact match against GET /api/events?search={artist}&start={today}; treat fuzzy hits as no-match for v1.
  • Notch cut-through uses a CSS mask (mask-composite: intersect), supported in current Safari/Chrome. If a target ever lacks it, the ticket simply loses the two notches — the dashed perforation still reads.
  • Server contract. Enrich each Playcut with one optional upcomingShow (or upcomingShows[] for the multi-date case); iOS renders purely from its presence and fields.
